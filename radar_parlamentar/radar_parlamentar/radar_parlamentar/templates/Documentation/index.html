A. HTML Structure

"Circle" is a fixed width (980px) html template. It is marked-up using HTML5 standard tags and attributes but works so nicely with older browser such as IE7. It is vertically divided into three main parts, header, content and footer. In this section I'll cover all these three main sections and their common sub sections.

Please take a look at the bottom picture and you'll have a complete idea of this template's HTML structure. And click the image to view the sections divided in the actual design.

Here we're going to guide the page structures, from top to bottom and left to right (where necessary).

Header (#header) Section

Header section is vertically divided into two sections one is the slider(#imageBox) and other one is the logo+menu(#navNLogo). They both absolutely positioned to support menu+logo at the top of the page instead of the slider.

There's additionally and element .bar floating just above the menu+logo container. This is the very top border (floating above slides or page header images).

  • Menu and Logo#navNLogo

    Actual menu and logo wrapped within a element .wrap to float them within the intended page width 980px and center to the page. The Logo (#logo) and Menu (nav) elements float side by side. Changing the logo is rather simplistic just point the image source to our own logo. Our actual Menu items are inside the ul#menu elements of the nav element. The menu structure is standard just use the first level li of ul#menu to place your parent menu items. And use nested ul element inside the first li element for your sub menus. Please the check the structure of the demonstration pages to better understand them.

    Just place an .active class to the appropriate active menu item to highlight them as active. You may need to adjust your header menu's left padding in-order to keep them on the correct order (aligned to the right). Just reduce or add some paddings from the #menu selector in the c. Menu section of the style.css file.

  • Slides and Header Images #imageBox

    We've placed our slides inside the Header section instead of the Content section for this theme's conveniyance. Please check the Slides+Header Image section for detail about their usage and avialble options.

Content section#content

All the actual page contents are inside of this section section#content. Please check pagewide guideline for individual page's settings and about their modifications.

Footer #footer

Our footer is vertically divided into two sub-sections and those are top .top and bottom .bottom. And here's their details:

  • top .top

    This section is for the individual footer widgets area. It has a first class child .wrap to wrap all the column inside a 980px width box and float them at the center of the page. You can use the default footer widgets already designed for you. Or can use any variations of them e.g. more or less columns. You can use the same column settings as in the footer as shown in the columns.html demonstration page for the contents section.

  • bottom .bottom

    The bottom is for the copyright notice and footer social links. You can use any kind of social links please refer to the Sources and Credit section for more about social icons and how you can get more.

D. Common Elements

Slides and Header Images

We can use slides on any page of this theme.

  • Slider on Homepage

    Circle has a total 7 slider variations. All the slides options and features are almost same as the plugin (slider scripts) default settings. You can change any of these sliders options like duration, effect, etc. Please open any slider script file and follow their site link from the top of those files. There you'll find some useful guidelines about tweaking those plugins.

    Remember we're using a full size image for our slides and header images. And we don't know that how many and what kind of large monitors our users are going to use to visit our site. So, we're placing our slides and header images through CSS. For any slides just take a new li element with appropriate date-slide attribute (as shown in the demonstration page index.html). Then search for @#@Slides inside the style.css file and add your slide images link as shown in there already. You can additionally add slide messages as shown in the demonstration and in order to do that just replace the demo contents with your own from inside the .slideMsgs container of every slide li items of the #slide element.

  • Slider on Other pages

    Using slides on other pages is very simple. Please check our Portfolio pages for demonstration. Everything goes just like our Homepage slides the difference is you can't place individual slides messages on other pages other pages and there will be no controlling (e.g. next or prev button) option.

    Use the settings shown on the portfolio pages for your individual pages header image slides. Follow these steps to add your own slides.

    1. We're going to place our slides inside the div.image element of the #imageBox element. Make sure there is a named .headerImages as it is controlling the height of our slides+images on every pages except homepage.
    2. Replace the default .portfolio class name with your own appropriate class name(s). And make sure the two class .image and .slides are intact to this element.
    3. Take a new li element inside the ul element of the .slides container. Add an appropriate class names to this li items. Then link the slide image from the style.css files @#@Other section.
    Remember you can only use individual slides messages on homepage slides.

  • Slide Scripts

    We're mostly using jQuery cycle plugin for our slides. You can change any settings of every slides from the init.js file's Site Initialization code section. For more available options (e.g. effects, duration, etc) of cycle plugin please visit jquery.malsup.com/cycle/.

    Beside cycle we're also using Nivo and Elastic sliding script. You'll find any home page slides scripts initialization code at the bottom of that slider page. So, you can use that index.html file and ignore the rest (index-* files).

H. Changing Theme Colors

Circle comes with premade five color variations. Even if you need more color variant so please first open any color css files from the files->codes->css->colors folder then replace link and header colors with your own colors. And remember the colors of the header of the slides messages and page title (the large title at every other pages than of homepage) are a little dark than of the general header color. You should check and match your darkness on these headings with your general header color. Follow these steps:

You can play with file index-color.psd to check how you're intended color will look by not harming the actual index.psd file.
  • Open the file orange-green.css file from inside the files->codes->css->colors folder. Take a new CSS document then copy everything from the orange-green.css file and paste them inside this new file. Name your file. Now replace the color #12a108 (of your file) with your own link color (You should find 4 repleaces).
  • Now find and replace the color #e94025 with your own heading color. (You should find 5 replaces). Then search for #dc3015 and replace this color with a slight darker version of your heading color.

Editing Graphics

  • Open the the elements.psd file from your psd folder. Add your link color to the colored item of the graphics except the circles. For the circles add a lighter version of your header color.
  • Save those items with save for "Wev and Devices.." option (as transparent png image) and place them inside the folders mentioned above those graphics (in elements.psd file).
  • Note there's a color code (-og og=orage-green) at the end of the each graphics (except blog types) linked from your custom color CSS file. I suggest you to add such extension to your every graphics you save from the Photoshop file. It will help you to keep the default graphics intact in the folders and you can use them later if you want.
    Also note that you don't need to slice any graphics as I've already sliced and named them appropriately for you! (Enable slicing option in Photoshop to check them). Just add your link colors to the colored one and hit "Save for Web and Devices.." also disable the background layer in-order to save as transparent png files.
  • Now replace the last -og extension from your color CSS file with your own color extension. And also replace the ../og/.. folder name to your own extension (find them at the bottom of the files->codes->css->colors->anycolor.css file).
  • Finally add the heading color to the background of the active menu item of the index.psd file. It has already got a sliced zone so just append your color extension and remember to disable only the inner glow filter then save for "Web and Devices.." and save it to the files->images folder.

Copy everything from this file and paste them directly at the very bottom of our style.css file and remember to remove the <link rel= "stylesheet" id="colors" href="files/codes/css/colors/[file].css" type="text/css" /> line from the bottom of your every page.

Now referesh and Congratulations! you're done. That was so quick, I think it should not take more than 15 minutes.

Using Color and Pattern Variations (Comes with this Theme)

The theme comes with lots of color and pattern variations. Please follow this steps in order to easily use any of the provided color or pattern variations.

  1. To Use Color Variations

    Simply copy your desired colors files css styles (find it files->codes->css->colors->colorfile.css) and then paste them at the bottom of our style.css file (find it files->codes->css->style.css).

  2. To Use Pattern Variations

    By default it has white pattern for header and content/body and dark one for footer (noise pattern). You can use white pattern for body and dark patterns for footer and both of them for header. To change header pattern First, locate #imageBox inside b. Slider + Page Msg section of style.css file. Then add a background color of #fff (if you use dark header background then use #222) then add your pattern file path. Additional (except default) pattern path should be ../../images->pattern->dark/light/your_pattern_graphic.png. Find your pattern file name by pointing your mouse over the pattern choosing thumb (find the name in the tooltip). Then select the file from inside the files->images->patterns->dark/light folder.

    Follow these same steps for content and footer patterns. For content add your pattern style to the body (at reset.css file) and footer add your pattern style to the #footer (at style.css file).

I. Others

a. Form Validation and Email

This theme is using jQuery tinyValidator plugin to validate form fields. It is a very light-weight validating library. I have developed it and it's free to use and share. To know how it works please open "init.js" file and in that file you'll find its working details.

In order to use php mail sender script to receive emails from your visitors just change the $to variable's value to your actual email address from inside mail.php. For details please read the inline comments inside this script file.

Fonts

This theme has been developed using three different fonts and those are: Museo, Helvetica Neue, and Lato. Among them I've only included Museo font family through CSS @font-face font embeding method. You'll find that font inside our assets->fonts directory. You'll find three different face of Museo (700,500,300) those are free for commercial or personal project. And by default this theme is using 700 and 500 (referenced bold and normal respectively in this theme). Additionally you'll find Museo 300 variants embeding function (code block) commented out (at the top of reset.css file) for your reference in-case you need them in your project.

Please don't get freezed with these two default fonts. You can use any fonts you like. But they works just find with this theme. To change your heading or body font face simply change them from the reset.css files heading and body font declaration block.
